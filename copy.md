# 3. 문제 제기 (수정)

### 섹션 제목 (기존 → 수정)

기존:
```text
혹시 지금도 이런 고민을 하고 있나요?
```

수정:
```text
원서 접수 전까지도
확신이 없는 이유
```

### 설명 (신규)

```text
성적표만으로는
어떤 대학이 나에게 유리한지 알 수 없기 때문입니다.
```

### 고민 카드 (기존 유지, 실제 댓글처럼 비정렬 배치)

> "제 성적으로 어디까지 지원할 수 있는지 모르겠어요."

> "논술은 자신 있는데 수능 최저가 불안해요."

> "대학마다 논술 유형이 달라서 뭘 준비해야 할지 모르겠어요."

> "경쟁률은 높은데 실제로 저한테 불리한 학교인지 판단이 안 돼요."

> "상향을 써야 할지, 안정 지원을 더 늘려야 할지 모르겠어요."

> "지원은 여섯 번뿐인데 하나라도 잘못 쓰면 너무 아깝잖아요."

(익명 라벨만 사용, 프로필 사진 없음. 실제 후기가 아니라면 **'수험생의 대표적인 고민을 재구성한 예시'**라고 표기)

### 연결 카피 (기존 유지)

논술 입시는 정보가 부족해서 어려운 것이 아닙니다.

정보는 많지만, **그 정보가 나에게 어떤 의미인지 판단하기 어렵기 때문입니다.**

논술핏은 흩어진 전형 정보를 학생 개인의 데이터와 연결해 실제 지원 결정에 필요한 답으로 바꿉니다.

# 4. 입력 화면 미리보기 (신규)

### 섹션 제목

```text
복잡한 서류 없이,
현재 성적만 입력하세요.
```

### 설명

```text
내신과 최근 모의고사 성적, 희망 계열을 입력하면
현재 지원 가능 범위부터 먼저 확인할 수 있습니다.
```

### 구성 — 왼쪽: 실제 입력 폼 목업

```text
내신 평균     3.4
국어         2등급
수학         4등급
영어         2등급
희망 계열     인문사회
```

### 구성 — 오른쪽: 안내 카드

```text
입력 예상 시간
약 3분

필수 입력 항목
6개

논술 경험이 없어도
기본 분석 가능
```

### 인터랙션

* 등급 선택 버튼 / 희망 계열 탭
* 입력 완료율 표시
* `분석 준비 완료` 상태 변화
* 필드 입력 시 오른쪽 요약 카드 즉시 갱신

# 5. 분석 엔진 다이어그램 (신규)

### 섹션 제목

```text
한 가지 점수로는
지원 전략을 만들 수 없습니다.
```

### 설명

```text
논술핏은 성적만 보는 대신,
학생과 대학 사이의 여러 조건을 함께 비교합니다.
```

### 다이어그램 구조

```text
학생 성적 · 논술 역량 · 대학별 전형 · 수능 최저
                    ↓
          논술핏 전략 분석 엔진
                    ↓
지원 가능 대학군 · 추천 이유 · 위험 요소 · 6개 원서 조합
```

### 변수 명칭 — 학생 친화적 표현으로 수정

| 내부 표현 | 랜딩 표현 |
|---|---|
| Essay Type | 나와 맞는 논술 유형 |
| SAT Min | 수능 최저 충족 가능성 |
| Student Record | 학생부가 얼마나 불리한지 |
| Portfolio Balance | 6장의 원서를 어떻게 나눌지 |

### 네 가지 분석 변수 상세 카피 (기존 → 수정)

**수능 최저**
기존: 수능 최저 안정도 → 수정: **수능 최저 충족 가능성**
설명: 최근 성적 흐름과 과목 조합을 바탕으로 대학별 최저 위험을 확인합니다.

**논술 유형**
기존: 논술 유형 적합도 → 수정: **나와 잘 맞는 논술 유형**
설명: 영어 제시문, 자료 해석, 수리 문항 등 학생의 강점과 대학별 출제 유형을 연결합니다.

**학생부**
기존: 학생부 실질 영향 → 수정: **학생부가 얼마나 불리한지**
설명: 명목 반영 비율이 아니라 등급 간 실제 감점 폭을 확인합니다.

**포트폴리오**
기존: 지원 포트폴리오 → 수정: **6장의 원서를 어떻게 나눌지**
설명: 상향·적정·안정 지원을 개별 대학이 아닌 하나의 조합으로 설계합니다.

### 디자인

* 연결선은 얇은 곡선, 스크롤 진입 시 선을 따라 빛이 이동
* 데이터 카드 순서대로 활성화, 중앙 엔진 카드에 분석 상태 애니메이션
* 결과 카드는 오른쪽에서 펼쳐지는 방식

LT학부

# 7. 성적 변화 시뮬레이션 (신규)

### 섹션 제목

```text
영어가 한 등급 오르면
지원 전략은 어떻게 달라질까요?
```

### 설명

```text
목표 등급을 조정해보고,
지원 가능한 대학군과 원서 조합의 변화를 확인해보세요.
```

### 구성 — 왼쪽: 성적 조정 UI

```text
영어 등급   2등급 → 1등급
수학 등급   4등급 → 3등급
```

### 구성 — 오른쪽: 즉시 반영되는 결과

```text
지원 가능 대학군      12개 → 17개
상향 지원 후보         2개 → 5개
수능 최저 안정 대학    6개 → 10개
```

### 전략 문장 자동 전환

변경 전:
```text
영어 최저 변동성 때문에 일부 상향 대학의 위험도가 높습니다.
```

변경 후:
```text
영어 1등급 확보 시 한국외대·성균관대 계열의 상향 지원 범위가 확대됩니다.
```

### 표현 주의사항

합격 확률처럼 보이는 과도한 수치는 피하고, 다음 표현만 사용합니다.

```text
지원 범위 확대 / 위험도 감소 / 최저 충족 대학 증가 / 상향 후보 추가
```

# 8. Before / After 학생 사례 (수정 — 변화 중심)

### 섹션 제목

```text
같은 평균 등급이어도
추천 결과는 달라집니다.
```

### 설명

```text
강점과 위험 요소가 다르면
같은 대학도 상향, 적정, 안정 분류가 달라질 수 있습니다.
```

### 사례 구조 (학생 A/B/C 탭 전환)

**학생 상황**
```text
내신 3.4
국어 2 / 수학 4 / 영어 2 / 탐구 3·3
```

**분석 전**
```text
중앙대, 외대, 건국대 정도를 모두 비슷하게 생각함
```

**논술핏 분석 후**
```text
상향: 중앙대
적정: 한국외대, 건국대
안정: 숭실대, 광운대
```

**분석으로 달라진 점**
```text
영어 제시문 적합도가 높아 한국외대 추천 순위 상승
수학 최저 변동성 때문에 중앙대는 상향으로 분류
학생부 영향이 작은 숭실대를 안정 카드로 추가
```

> 탭 전환 시 성적 프로필, 추천 대학 조합, 핵심 위험, 다음 행동, Before/After 문장이 함께 변경됨.

### 기존 사례 A/B (원문 유지, 보조 자료로 활용)

**사례 A** — 내신 2.8, 국어2/수학4/영어2/탐구3, 강점: 영어 독해·비교형 답안, 약점: 수리 문항
> "학교 이름만 보면 건국대와 숭실대가 끌리는데 저한테 실제로 맞는지 모르겠어요."
적극 검토: 한국외국어대학교, 동국대학교, 숭실대학교 / 조건부 검토: 건국대 상경계열, 경희대 사회계열

**사례 B** — 내신 3.6, 국어3/수학3/영어4/탐구2, 강점: 도표 해석, 약점: 영어 제시문
> "경쟁률이 낮은 대학 위주로 지원하면 합격 가능성이 높아질까요?"
핵심 판단: 경쟁률보다 영어 최저와 제시문 여부가 더 큰 위험 요인

하단 CTA: **내 성적으로 같은 방식의 분석 받아보기**

# 12. FAQ (기존 유지)

**논술핏은 합격 확률을 알려주는 서비스인가요?**
단순한 합격률 예측 서비스가 아닙니다. 성적, 논술 역량과 대학별 전형 조건을 함께 분석해 지원 적합도와 위험 요소, 지원 조합을 제공합니다.

**합격을 보장하나요?**
아니요. 입시는 실제 시험 결과, 경쟁률과 대학의 평가에 따라 달라집니다.

**논술 경험이 없어도 사용할 수 있나요?**
네. 현재 성적을 기반으로 지원 가능 범위와 전형 조건을 분석할 수 있습니다.

**수능 최저가 없는 대학도 분석하나요?**
네. 논술 적합도, 학생부 반영 방식, 경쟁률과 준비 부담 등 다른 요소의 비중을 높여 분석합니다.

**추천 점수가 합격 가능성을 의미하나요?**
아니요. 학생에게 얼마나 전략적으로 적합한지를 나타내는 우선순위 지표입니다.

**대학 모집요강이 변경되면 어떻게 되나요?**
공식 전형계획과 모집요강 변경 사항을 기준으로 데이터를 갱신합니다.

**분석 결과는 얼마나 걸리나요?**
일반적으로 몇 분 안에 전략 리포트를 확인할 수 있도록 구성합니다.

**내 정보는 어떻게 사용되나요?**
입력한 성적과 논술 데이터는 지원 전략 분석과 리포트 제공을 위해서만 사용합니다.

랜딩은 사용자가 10초 안에 아래를 이해해야 합니다.

```text
무슨 서비스인가?        논술 지원 전략 분석 서비스
무엇을 분석하는가?      성적, 논술 유형, 수능 최저, 대학 전형
무엇을 받는가?          지원 가능 대학군과 6개 원서 조합
무엇이 다른가?          추천 이유와 위험 요소까지 설명함
얼마나 걸리는가?        약 3분
다음 행동은?            무료 분석 시작
```

시각적 기준:

* 제목이 모바일에서 어색하게 잘리지 않음
* 카드 높이가 들쭉날쭉하지 않음
* 모든 섹션이 같은 카드 디자인을 반복하지 않음
* 실제 제품 화면(입력→분석→리포트→시뮬레이션)이 랜딩의 중심임
* 인터랙션이 장식이 아니라 제품 이해를 돕고 있음
* 가짜 통계처럼 보이는 수치가 없음 (`샘플 분석 결과` 라벨 필수)
* 모바일에서 CTA가 항상 접근 가능함
