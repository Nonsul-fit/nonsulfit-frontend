좋아. 지금 랜딩을 기준으로 보면 방향은 명확합니다.

핵심은 단순히 섹션을 더 붙이는 게 아니라,

> **"설명하는 랜딩"에서 "직접 체험시키는 제품형 랜딩"으로 바꾸는 것**

입니다.

아래 수정안은 앞에서 말한 6개 추가 기능과 카피, 레이아웃, 모바일, 인터랙션, 시각적 고급화까지 전부 합친 최종 개편안입니다.

---

# 1. 전체 개편 목표

현재 랜딩의 장점은 이미 충분합니다.

* 브랜드 컬러가 안정적임
* 리포트 카드가 보기 좋음
* 정보 구조가 이해 가능함
* 학생 대상 서비스라는 점이 드러남
* 인터랙션이 기본적으로 존재함

다만 아직은 다음 인상이 남아 있습니다.

> "잘 만든 입시 SaaS 랜딩페이지"

개편 후에는 다음 인상을 목표로 해야 합니다.

> "실제 분석 엔진과 완성된 제품이 존재하는 논술 전략 플랫폼"

이를 위해 세 가지를 강화합니다.

### 제품성

입력, 분석, 결과, 시뮬레이션을 실제로 체험하게 합니다.

### 신뢰성

결과가 어떻게 생성되는지와 추천 이유를 시각적으로 설명합니다.

### 디자인 완성도

텍스트 줄바꿈, 카드 밀도, 배경 리듬, 모션, 모바일 구성을 전면 정제합니다.

# 2. 최종 랜딩페이지 구조


지금보다 섹션 수는 조금 늘지만, 단순 텍스트 섹션이 아니라 대부분 제품 시연형이므로 체감 길이는 오히려 덜 지루합니다.

# 3. Hero 전면 개편

## 현재 문제

현재 Hero는 메시지는 좋지만 다음이 아쉽습니다.

* 제목이 길어서 모바일 줄바꿈이 어색할 수 있음
* 오른쪽 리포트가 정적인 목업처럼 보임
* 입력부터 결과까지 어떻게 이어지는지 알 수 없음
* 수치가 실제 필드인지 데모인지 불분명함
* 핵심 차별점이 본문 안에 묻힘

## 추천 Hero 카피

### 상단 라벨

```text
논술 지원 전략 분석 리포트
```

### 메인 헤드라인

```text
논술 지원,
이제는 감이 아니라
근거입니다.
```

이 카피가 지금의 긴 문장보다 브랜드 메시지가 강합니다.

### 서브카피

```text
성적, 논술 역량, 수능 최저, 대학별 전형을 함께 분석해

지원 가능한 대학과
왜 그 대학이 유리한지,
무엇을 먼저 보완해야 하는지까지 알려드립니다.
```

다만 HTML에서는 데스크톱과 모바일 줄바꿈을 다르게 설계해야 합니다.

### CTA

주 CTA:

```text
내 지원 전략 무료로 확인하기
```

보조 CTA:

```text
실제 리포트 먼저 보기
```

보조 문구:

```text
약 3분 소요 · 추천 근거 포함 · 6개 지원 조합 제공
```

## Hero 디자인

오른쪽을 단일 카드가 아니라 **3단계 미니 제품 데모**로 바꿉니다.

```text
성적 입력
   ↓
전형 분석 중
   ↓
추천 대학 6개 생성
```

처음에는 입력 카드가 보이고, 2초 후 분석 상태, 다시 2초 후 리포트 결과로 바뀌게 합니다.

### 플로팅 보조 카드

리포트 주변에 작은 카드 2~3개를 배치합니다.

```text
수능 최저 충족 가능성 84%
```

```text
지원 가능 대학군 12개
```

```text
상향 2 · 적정 2 · 안정 2
```

단, 모든 숫자는 반드시 `샘플 분석 결과`라는 작은 라벨을 포함해야 합니다.

# 4. 추가 기능 1 — 실제 입력 화면 미리보기

이 기능의 목적은 사용 장벽을 낮추는 것입니다.

사용자는 결과보다 먼저 이런 걱정을 합니다.

> "뭘 얼마나 입력해야 하지?"

## 구성

왼쪽:

실제 분석 입력 폼 목업

```text
내신 평균
3.4

국어
2등급

수학
4등급

영어
2등급

희망 계열
인문사회
```

오른쪽:

```text
입력 예상 시간
약 3분

필수 입력 항목
6개

논술 경험이 없어도
기본 분석 가능
```

## 인터랙션

* 등급 선택 버튼
* 희망 계열 탭
* 입력 완료율 표시
* `분석 준비 완료` 상태 변화
* 필드 입력에 따라 오른쪽 요약 카드가 즉시 변경

## 카피

섹션 제목:

```text
복잡한 서류 없이,
현재 성적만 입력하세요.
```

설명:

```text
내신과 최근 모의고사 성적, 희망 계열을 입력하면
현재 지원 가능 범위부터 먼저 확인할 수 있습니다.
```

# 5. 추가 기능 2 — 분석 엔진 다이어그램

이 섹션은 논술핏의 기술적 차별성과 신뢰를 보여줍니다.

## 구조

중앙에 `논술핏 전략 분석 엔진`을 배치하고, 네 방향에서 데이터가 들어오게 합니다.

```text
학생 성적
논술 역량
대학별 전형
수능 최저
     ↓
논술핏 전략 분석 엔진
     ↓
지원 가능 대학군
추천 이유
위험 요소
6개 원서 조합
```

## 디자인

* 연결선은 얇은 곡선
* 스크롤 진입 시 선을 따라 빛이 이동
* 각 데이터 카드는 순서대로 활성화
* 중앙 엔진 카드에는 짧은 분석 상태 애니메이션
* 결과 카드가 오른쪽에서 펼쳐지는 방식

## 카피

제목:

```text
한 가지 점수로는
지원 전략을 만들 수 없습니다.
```

설명:

```text
논술핏은 성적만 보는 대신,
학생과 대학 사이의 여러 조건을 함께 비교합니다.
```

각 변수 명칭은 학생 친화적으로 씁니다.

| 내부 표현             | 랜딩 표현           |
| ----------------- | --------------- |
| Essay Type        | 나와 맞는 논술 유형     |
| SAT Min           | 수능 최저 충족 가능성    |
| Student Record    | 학생부가 얼마나 불리한지   |
| Portfolio Balance | 6장의 원서를 어떻게 나눌지 |

# 6. 추가 기능 3 — 실제 대시보드형 리포트

이 부분이 랜딩의 핵심 제품 섹션이 됩니다.

현재 카드 형태보다 훨씬 실제 서비스처럼 보여야 합니다.

## 레이아웃

### 좌측 사이드바

```text
학생 요약
지원 가능 대학군
추천 원서 조합
대학별 상세 분석
수능 최저 분석
다음 행동
```

### 중앙

대학 카드 6개

* 상향 2개
* 적정 2개
* 안정 2개

각 카드에는 다음을 넣습니다.

```text
한국외국어대학교
LT학부

논술 적합도       높음
최저 충족 가능성   보통
학생부 영향        낮음

왜 추천하나요?
영어 제시문 강점을 가장 잘 활용할 수 있는 적정 지원 카드
```

### 우측 또는 상세 패널

선택한 대학의 상세 내용

```text
추천 이유
주의 요소
유리한 전형 요소
보완 우선순위
최근 분석 기준일
```

## 상향·적정·안정 구분

색상만 다르게 하지 말고 톤과 구조를 함께 구분합니다.

### 상향

* 옅은 주황 배경
* 상승 화살표 아이콘
* `도전 가능` 배지
* 위험 요소 우선 노출

### 적정

* 파란색 배경
* `가장 추천` 배지
* 추천 이유를 가장 크게 노출

### 안정

* 옅은 초록 배경
* 방패 아이콘
* 최저 안정성과 학생부 영향 강조

# 7. 추가 기능 4 — 성적 변화 시뮬레이션

이 기능은 논술핏만의 가장 기억에 남는 랜딩 인터랙션이 될 수 있습니다.

## 구성

왼쪽에 성적 조정 UI를 둡니다.

```text
영어 등급
2등급 → 1등급

수학 등급
4등급 → 3등급
```

슬라이더 또는 선택 버튼 방식이 좋습니다.

오른쪽 결과가 즉시 바뀝니다.

```text
지원 가능 대학군
12개 → 17개

상향 지원 후보
2개 → 5개

수능 최저 안정 대학
6개 → 10개
```

그리고 하단에 전략 문장이 바뀝니다.

변경 전:

```text
영어 최저 변동성 때문에 일부 상향 대학의 위험도가 높습니다.
```

변경 후:

```text
영어 1등급 확보 시 한국외대·성균관대 계열의 상향 지원 범위가 확대됩니다.
```

## 섹션 카피

제목:

```text
성적 하나가 바뀌면
지원 전략은 어떻게 달라질까요?
```

설명:

```text
목표 등급을 조정해보고,
지원 가능한 대학군과 원서 조합의 변화를 확인해보세요.
```

## 중요한 주의

합격 확률처럼 보이는 과도한 수치는 피합니다.

추천 표현:

```text
지원 범위 확대
위험도 감소
최저 충족 대학 증가
상향 후보 추가
```

# 8. 추가 기능 5 — Before / After 학생 사례

지금의 학생 A/B/C 방식은 유지하되, 설명형이 아니라 변화 중심으로 바꿉니다.

## 사례 구조

### 학생 상황

```text
내신 3.4
국어 2
수학 4
영어 2
탐구 3 / 3
```

### 분석 전

```text
중앙대, 외대, 건국대 정도를 모두 비슷하게 생각함
```

### 논술핏 분석 후

```text
상향
중앙대

적정
한국외대
건국대

안정
숭실대
광운대
```

### 분석으로 달라진 점

```text
영어 제시문 적합도가 높아 한국외대 추천 순위 상승

수학 최저 변동성 때문에 중앙대는 상향으로 분류

학생부 영향이 작은 숭실대를 안정 카드로 추가
```

## 인터랙션

학생 A/B/C를 누르면 다음이 함께 변경됩니다.

* 성적 프로필
* 추천 대학 조합
* 핵심 위험
* 다음 행동
* Before/After 문장

## 카피

제목:

```text
같은 평균 등급이어도
추천 결과는 달라집니다.
```

설명:

```text
강점과 위험 요소가 다르면
같은 대학도 상향, 적정, 안정 분류가 달라질 수 있습니다.
```

# 11. 텍스트가 잘리지 않도록 하는 디자인 규칙

이 부분은 반드시 시스템적으로 적용해야 합니다.

## 제목

```css
.heading {
  text-wrap: balance;
  word-break: keep-all;
  line-height: 1.08;
  letter-spacing: -0.04em;
}
```

* 데스크톱 최대 폭: `12~16ch`
* 모바일 최대 폭: `100%`
* 제목 안의 핵심 줄바꿈은 직접 지정
* 모바일과 데스크톱 줄바꿈을 따로 처리

예:

```html
<h1>
  논술 지원,<br>
  이제는 감이 아니라<br>
  <span>근거입니다.</span>
</h1>
```

## 본문

```css
.body-copy {
  max-width: 62ch;
  line-height: 1.75;
  word-break: keep-all;
}
```

## 카드 제목

```css
.card-title {
  min-height: 3.2rem;
  text-wrap: balance;
  word-break: keep-all;
}
```

## 카드 설명

```css
.card-description {
  min-height: 5.2rem;
  line-height: 1.65;
}
```

가능하면 카드 안 설명은 3~4줄 이내로 제한합니다.

## 긴 대학명

대학명은 말줄임표로 자르면 안 됩니다.

```css
.school-name {
  word-break: keep-all;
  overflow-wrap: normal;
}
```

한 줄에 못 들어가면 자연스럽게 2줄을 허용합니다.

# 12. 디자인 고급화

## 12.1 배경 리듬

현재의 흰색/회색 반복에서 벗어나 다음처럼 구성합니다.

| 섹션       | 배경              |
| -------- | --------------- |
| Hero     | 밝은 그리드 + 블루 글로우 |
| 문제 제기    | 흰색              |
| 입력 미리보기  | 연한 블루           |
| 엔진 다이어그램 | 흰색              |
| 리포트 데모   | 짙은 네이비          |
| 시뮬레이션    | 연한 회색           |
| 사례       | 흰색              |
| 비교       | 연한 블루           |
| CTA      | 네이비             |

리포트 데모를 짙은 배경에 배치하면 가장 중요한 제품 화면이 강하게 강조됩니다.

---

## 12.2 카드 체계

모든 카드가 같은 스타일이면 단조롭습니다.

카드 유형을 세 가지로 나눕니다.

### 정보 카드

* 얇은 테두리
* 흰 배경
* 최소 그림자

### 결과 카드

* 연한 컬러 배경
* 강조된 점수
* 배지
* 상향·적정·안정별 색상 구분

### 상세 카드

* 더 큰 여백
* 설명 위주
* 아이콘 최소화
* 핵심 문장 강조

---

## 12.3 그림자

현재 그림자가 조금 전형적인 SaaS 느낌입니다.

다음처럼 단계화합니다.

```text
기본 카드
0 8px 24px rgba(15, 52, 105, 0.06)

중요 카드
0 18px 50px rgba(15, 52, 105, 0.10)

Hero 리포트
0 30px 90px rgba(15, 52, 105, 0.16)
```

모든 카드에 그림자를 넣지 말고, 핵심 요소에만 사용합니다.

---

## 12.4 Border Radius

현재 너무 많은 요소가 비슷하게 둥글면 장난감 같은 느낌이 날 수 있습니다.

추천 기준:

```text
작은 버튼       12px
입력 필드       14px
일반 카드       20px
중요 카드       28px
Hero 대시보드   32px
```

---

## 12.5 아이콘

Material Symbols를 계속 사용해도 되지만, 제품 전체에서 한 세트로 통일해야 합니다.

더 세련된 방향은:

* Lucide
* Phosphor
* Heroicons

중 하나로 통일하는 것입니다.

아이콘은 장식이 아니라 의미 전달이 필요한 곳에만 사용합니다.

* 학생부 영향

# 14. 모바일 전용 재설계

모바일은 데스크톱을 단순 축소하면 안 됩니다.

## 모바일 순서

```text
Hero 카피
CTA
핵심 결과 3개
리포트 카드
입력 미리보기
문제 제기
분석 변수
리포트 상세
시뮬레이션
사례
비교
FAQ
최종 CTA
```

## 모바일에서 제거하거나 축소할 요소

* 복잡한 엔진 다이어그램은 세로 스텝형으로 변경
* 사이드바 대시보드는 상단 탭으로 변경
* 비교 테이블은 카드 비교 형태로 변경
* 레이더 차트는 단순 막대 그래프로 변경
* 플로팅 카드는 1개만 유지
* 긴 설명은 접기 제공

## 모바일 타이포

```text
Hero 제목       40~44px
섹션 제목       30~34px
카드 제목       18~20px
본문            15~16px
보조 문구       13~14px
```

모바일에서는 `word-break: keep-all`이 특히 중요합니다.

# 17. 최종 완성 기준

개편 후 랜딩은 사용자가 10초 안에 다음을 이해해야 합니다.

```text
무슨 서비스인가?
논술 지원 전략 분석 서비스

무엇을 분석하는가?
성적, 논술 유형, 수능 최저, 대학 전형

무엇을 받는가?
지원 가능 대학군과 6개 원서 조합

다른 서비스와 무엇이 다른가?
추천 이유와 위험 요소까지 설명함

얼마나 걸리는가?
약 3분

다음에 무엇을 해야 하는가?
무료 분석 시작
```

그리고 시각적으로는 다음 기준을 만족해야 합니다.

* 제목이 모바일에서 어색하게 잘리지 않음
* 카드 높이가 들쭉날쭉하지 않음
* 모든 섹션이 같은 카드 디자인을 반복하지 않음
* 실제 제품 화면이 랜딩의 중심임
* 인터랙션이 장식이 아니라 제품 이해를 돕고 있음
* 가짜 통계처럼 보이는 수치가 없음
* 모바일에서 CTA가 항상 접근 가능함

이 개편안대로 진행하면 디자인 관점에서는 **9.5점 전후**, 실제 제품 연결과 신뢰 요소까지 갖추면 **9.7점 이상**을 노릴 수 있는 구조입니다.
